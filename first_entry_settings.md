# Техническое задание: First Entry - Settings

## Общее описание
После завершения онбординга пользователь проходит процесс первичной настройки приложения для отслеживания элайнеров. Это многошаговый процесс, состоящий из 8 экранов, где пользователь вводит основные параметры своего лечения.

---

## Экран 1: Приветствие и начало настройки

### 1. Назначение экрана
Приветственный экран, который открывает процесс первичной настройки приложения после онбординга. Информирует пользователя о предстоящих шагах настройки лечения.

### 2. Основные компоненты экрана
- **Заголовок приветствия**: Текст "Welcome!" в верхней части экрана
- **Описание процесса**: Текст "Let's go through a few steps to set up your treatment" в центральной части
- **Кнопка "Restore"**: В левой части нижней области экрана
- **Индикатор прогресса**: Ряд из 8 точек в центральной части нижней области, где первая точка активна
- **Кнопка "Next"**: В правой части нижней области экрана

### 3. Пользовательские сценарии
**Основной сценарий:**
1. Пользователь видит приветствие и описание процесса
2. Нажимает кнопку "Next" для перехода к следующему шагу

**Альтернативный сценарий:**
1. Пользователь нажимает "Restore" для восстановления предыдущих настроек (если доступно)

### 4. Состояния экрана
- **Обычное состояние**: Отображение всех элементов интерфейса
- **Состояние загрузки**: При нажатии "Next" или "Restore" (если требуется обработка)

---

## Экран 2: Общее количество элайнеров

### 1. Назначение экрана
Пользователь вводит общее количество элайнеров в своем курсе лечения.

### 2. Основные компоненты экрана
- **Заголовок**: Текст "Total number of aligners" в центральной части экрана
- **Поле ввода**: Иконка карандаша и число "30" в центральной части экрана
- **Кнопка "Cancel"**: В левой части нижней области
- **Индикатор прогресса**: Ряд из 7 точек, где 4-я точка активна
- **Кнопка "Next"**: В правой части нижней области

### 3. Пользовательские сценарии
**Основной сценарий:**
1. Пользователь видит предустановленное значение "30"
2. Нажимает на поле ввода для редактирования
3. Вводит корректное количество элайнеров
4. Нажимает "Next" для продолжения

**Альтернативные сценарии:**
1. Пользователь оставляет значение по умолчанию
2. Пользователь нажимает "Cancel" для возврата к предыдущему экрану

### 4. Состояния экрана
- **Обычное состояние**: Отображение поля ввода с числом
- **Состояние редактирования**: Активное поле ввода с клавиатурой
- **Состояние ошибки**: При вводе некорректного значения (отрицательное число, ноль)

## Экран 4: Тип элайнера

### 1. Назначение экрана
Пользователь выбирает тип прозрачного элайнера, который используется в его лечении.

### 2. Основные компоненты экрана
- **Заголовок**: Текст "Type of clear aligner for your treatment" в центральной части
- **Поле выбора**: Иконка карандаша и текст "3M Clarity" в центральной части
- **Кнопка "Cancel"**: В левой части нижней области
- **Индикатор прогресса**: Ряд из 8 точек, где 5-я точка активна
- **Кнопка "Next"**: В правой части нижней области

### 3. Пользовательские сценарии
**Основной сценарий:**
1. Пользователь видит предустановленный тип "3M Clarity"
2. Нажимает на поле для выбора другого типа
3. Выбирает нужный тип элайнера из списка
4. Нажимает "Next"

**Альтернативные сценарии:**
1. Пользователь оставляет предустановленный тип
2. Пользователь выбирает "Другой" и вводит свой тип

### 4. Состояния экрана
- **Обычное состояние**: Отображение выбранного типа
- **Состояние выбора**: Открытый список доступных типов элайнеров
- **Состояние ввода**: Поле для ввода пользовательского типа

Пользователь решает, включать ли автоматическое резервное копирование данных.

- **Кнопка "Restore"**: В левой части нижней области

## Общие принципы навигации

### Логика перехода между экранами
1. **Последовательный переход**: Пользователь проходит экраны в строгом порядке от 1 до 8
2. **Кнопка "Next"**: Переход к следующему экрану после корректного ввода данных
3. **Кнопка "Cancel"**: Возврат к предыдущему экрану (кроме первого экрана)
4. **Кнопка "Done"**: Завершение процесса настройки на последнем экране
5. **Кнопка "Restore"**: Восстановление предыдущих настроек (если доступно)

### Валидация данных
- Все числовые поля проверяются на корректность ввода
- Номер текущего элайнера не может превышать общее количество элайнеров
- Количество дней ношения не может быть отрицательным
- Время напоминания должно быть в допустимом формате

### Обработка ошибок
- Отображение сообщений об ошибках при некорректном вводе
- Возможность исправления введенных данных
- Предотвращение перехода к следующему экрану при наличии ошибок
